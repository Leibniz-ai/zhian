# 翔安区治安防控综合平台流程图方案

## 可直接给墨刀的提示词

请帮我生成一张“翔安区治安防控综合平台项目流程图”，整体采用横向流程图布局，参考政务科技风、浅色大屏汇报风格，背景以浅蓝白为主，模块分组清晰，连线规整，不要出现节点重叠。

设计要求：
- 画布为横向 16:9 或超宽画布。
- 整体视觉干净、专业、偏政府项目汇报风格。
- 一级流程从左到右推进。
- 不同阶段用不同颜色虚线分组框：
  - 蓝色：前置配置与智能感知
  - 紫色：告警分级与复核
  - 绿色：处置闭环与持续优化
- 开始节点使用深蓝色圆角胶囊。
- 判断节点“是否属实？”使用橙色菱形。
- 每个流程卡片使用白底、细描边、轻阴影，标题加粗。
- 连线尽量使用直线或直角线，避免杂乱弯折。
- 文案不要太长，每个卡片控制在 2 到 4 行。

请按以下业务结构生成：

1. 开始

2. 系统配置中心（蓝色分组）
   - 区域配置
     说明：启用重点防区，决定哪些街道、网格和重点场景进入实战布控与态势展示。
   - 摄像头点位配置
     说明：绑定视频通道、在线状态、核心点位和算法归属，形成可感知的视频资源池。
   - 规则与 ROI 配置
     说明：按场景设置 ROI 围栏、聚集人数阈值和持续时长阈值，用于突出重点区域并减少误报。

3. 智能感知与态势展示（蓝色分组）
   - 视频流实时接入
     说明：已启用防区的监控通道持续送流，作为聚集、打架、涉稳和异常行为识别的实时输入。
   - AI 场景识别
     子项：涉稳/讨薪聚集识别、多人异常聚集识别、打架斗殴/肢体冲突识别、偷盗/拉车门等异常行为链识别。
   - 事件指标提取
     说明：抽取防区、点位、时间、人数、ROI 密度、滞留时长、冲突标签、置信度和短视频片段。
   - 态势大屏与预警流
     说明：GIS 地图、防区状态和实时预警流联动展示，值班员点击告警即可进入复核处置页面。

4. 统一事件对象（绿色过渡节点或独立卡片）
   - 事件标准化汇总
     说明：把视频片段、识别结果、关键指标、防区位置、告警等级和处置上下文整理成统一事件。
   - 实时智能预警流
     说明：在态势大屏持续展示未处置告警，支持按防区查看并一键打开复核窗口。

5. 告警分级与极速复核（紫色分组）
   - 等级判断与入口分流
     说明：P0 进入极速复核台，P1 进入线索复核台。
   - P0 极速复核台
     说明：短视频优先展示；聚集场景显示人数、ROI 密度、持续时长；打架场景显示冲突标签、最近警力、处置建议；确认属实后再派警。
   - P1 线索复核台
     说明：查看片段和基础指标；适合拉车门、异常滞留等线索场景；确认属实则转警，否则回流误报样本。
   - 复核输出
     说明：统一输出“确认属实并处置”或“标记误报回流”。

6. 判断节点
   - 是否属实？

7. 若否，进入绿色分支
   - 误报回流 / 样本优化
     说明：将正常活动、非警情视频标记为误报样本，回流算法侧补充负样本，降低同类场景重复告警。

8. 若是，进入绿色分支“处置闭环与持续优化”
   - 联动处置
     说明：派发巡逻组、网格员、高音喇叭、入口管控、现场隔离等动作。
   - 历史处置记录
     说明：沉淀告警触发、确认、派单、解除全过程，以及证据、指挥员、时间线和备注信息。
   - 复盘报告
     说明：P0 场景自动生成复盘初稿，后续补充现场反馈、发现和改进建议。
   - 规则回写优化
     说明：将复盘结论回写到 ROI、阈值、SOP 和布控策略，形成下一轮更精准的防区布控。

9. 在流程图底部增加一条闭环回路箭头
   - 从“规则回写优化”回到“系统配置中心”
   - 表达“误报样本、处置记录、复盘结论持续反哺前置配置”

10. 结束
   - 可选：如果画面足够，可在最右下角或闭环末端增加“持续优化运行”作为结束语，而不是单独大节点。

请输出时注意：
- 节点之间留足间距。
- 文案以简洁、汇报化表达为主。
- 优先突出“视频感知 -> 告警复核 -> 联动处置 -> 记录复盘 -> 配置优化”的闭环。
- 不要把所有信息塞得过满，重点体现层级和主流程。

## 节点顺序简表

`开始`
-> `系统配置中心`
-> `智能感知与态势展示`
-> `统一事件对象`
-> `告警分级与极速复核`
-> `是否属实？`

分支一：
`否`
-> `误报回流 / 样本优化`
-> `规则回写优化`
-> 回到 `系统配置中心`

分支二：
`是`
-> `联动处置`
-> `历史处置记录`
-> `复盘报告`
-> `规则回写优化`
-> 回到 `系统配置中心`
